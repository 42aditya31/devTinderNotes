It will be - what do you mean by Waterfall Model: - Requirements(Project Manager + help of designer) --> Design(Senior Engineer/ Tech lead ) --> Development(SDE1, SDE2) --> Testing(SDET) ---> Deployment(DevOps + dev team ) --> Maintainance (whole cycle agian ) - what do mean by monolith vs microservices: - Monolith: A single, unified application where all components are interconnected and interdependent. - Microservices: An architectural style that structures an application as a collection of loosely coupled services, each responsible for a specific business capability. Each service can be developed, deployed, and scaled independently. Monolith vs Microservices: - Monolith: - Single codebase - Tight coupling of components - Easier to develop and deploy initially - Easier to develop initially - Harder to scale and maintain as it grows - Deployment requires the entire application to be redeployed - Microservices: - Multiple codebases, each for a service - Loose coupling of components - More complex to develop initially - Easier to scale and maintain as each service can be managed independently - Deployment can be done for individual services without affecting the entire application -Explain all the parameters what happend in the monolith vs microservices: - Dev speed, Code Repo , Scalability, Deployment, Team Structure, Testing, Maintainability, Performance, Technology Stack, Communication, Fault Isolation, Infra Cost, Complexity, Ownership, Rewamps, debugging , dev experience.
